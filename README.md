# Детекция курящих людей (YOLO11)

Система автоматической детекции курящих людей на изображениях с камер видеонаблюдения. Учебная практика, направление — компьютерное зрение.

## Задача

**Вход:** изображение с камеры видеонаблюдения.
**Выход:** координаты ограничивающих прямоугольников (bounding box) вокруг курящих людей, сигарет, вейпов и дыма.

**Ключевое требование:** система должна отличать курящего человека от некурящего.
- Человек не курит → объект не детектируется.
- Человек закурил → система фиксирует факт курения.

Это требование напрямую определило методологию разметки (см. ниже) — модель обучается находить не любого человека, а только того, кто активно курит.

Дополнительное условие: валидация модели проводится на фотографиях, где человек находится не ближе 10–15 метров от камеры — это делает объекты `cigarette`, `vape` и `smoke` особенно мелкими и сложными для детекции.

## Классы

| Класс | Описание |
|---|---|
| `person` | Курящий человек (только активно курящие люди размечены — некурящие в кадре не обводятся) |
| `cigarette` | Сигарета |
| `vape` | Электронная сигарета / вейп |
| `smoke` | Дым |

## Технологии

- **[Ultralytics YOLO11](https://docs.ultralytics.com/)** — одноэтапный детектор объектов, версия YOLO11s (также рассматривалась YOLO11m)
- **[CVAT](https://www.cvat.ai/)** — разметка собственного набора изображений (bounding box, экспорт в формате YOLO 1.1)
- **[Roboflow](https://roboflow.com/)** — источник дополнительных открытых датасетов (опционально, для расширения обучающей выборки)
- **Python 3 + Google Colab** — среда обучения модели, GPU NVIDIA T4 (бесплатный тариф)

## Структура датасета

Стандартная структура YOLO:

```
my_dataset/
├── images/
│   ├── train/
│   ├── val/
|   └── test/
├── labels/
│   ├── train/
│   └── val/
└── data.yaml
```

Пример `data.yaml`:
```yaml
path: /content/drive/MyDrive/smoking_detection/my_dataset
train: images/train
val: images/val

names:
  0: person
  1: cigarette
  2: vape
  3: smoke
```

Каждому изображению `images/<split>/<name>.jpg` соответствует файл разметки `labels/<split>/<name>.txt` (тот же `<name>`, формат YOLO: `class x_center y_center width height`, координаты нормализованы 0–1).

## Методология разметки

Разметка выполнялась вручную в CVAT:
1. Загрузить изображения в задачу (Task) в CVAT.
2. Настроить классы: `person`, `cigarette`, `vape`, `smoke` (тип — rectangle).
3. Обвести рамкой **только курящего человека** — некурящие люди в кадре намеренно не размечаются. Раз YOLO учится строго по размеченным объектам, непомеченные люди становятся для модели частью фона, и на инференсе модель, как правило, не выдаёт ложных рамок на людях без сигареты.
4. Отдельно обвести сигарету/вейп/дым, если видны в кадре.
5. Экспортировать: `Menu → Export job dataset → формат YOLO 1.1`. Если используется бесплатный облачный CVAT — тумблер **Save images** нужно выключить (экспорт изображений вместе с разметкой недоступен на бесплатном плане), и просто сопоставить скачанные `.txt` с уже имеющимися локально исходными изображениями по имени файла.

## Обучение модели

Обучение производится в Google Colab (см. `smoking_detection.ipynb` в репозитории):

```python
from ultralytics import YOLO

model = YOLO('yolo11s.pt')
results = model.train(
    data='path/to/data.yaml',
    epochs=150,
    imgsz=1280,       # важно для мелких объектов (сигарета/дым на дистанции)
    batch=16,
    device=0,
    patience=30,       # ранняя остановка при отсутствии улучшений
)
```

`imgsz=1280` — значимый параметр: при стандартных 640px модель плохо видит мелкие объекты вроде сигареты на дистанции 10–15 м.

### Основные шаги ноутбука
1. Установка `ultralytics` (и `roboflow`, если используются внешние датасеты).
2. Подключение Google Drive (датасет и результаты хранятся там между сессиями Colab).
3. (Опционально) скачивание/слияние дополнительных датасетов — классы сопоставляются **по названию**, а не по числовому индексу, поскольку разные источники используют разную нумерацию.
4. Диагностика датасета (проверка количества изображений/разметки, отсутствие пересечений train/val).
5. Обучение, валидация, инференс на тестовых изображениях.

Если сессия Colab обрывается на середине обучения — прогресс не теряется, можно продолжить:
```python
model = YOLO('runs/.../weights/last.pt')
results = model.train(resume=True)
```

## Результаты

Обучение YOLO11s, 65 из 150 эпох (остановка по early stopping), разрешение 1280px, ~5 часов на GPU T4.

| Класс | mAP50 |
|---|---|
| person | 0.620 |
| cigarette | 0.138 |
| smoke | 0.095 |
| vape | 0.000 |

**Наблюдения:**
- `person` детектируется уверенно — крупный, хорошо различимый объект с достаточным числом обучающих примеров (335 инстансов).
- `cigarette` и `smoke` показывают более скромный результат — типично для мелких объектов, занимающих малую долю кадра.
- `vape` не был выучен вовсе — всего 31 размеченных примеров в датасете, этого недостаточно для обучения нейросети на отдельный класс.

Полные графики обучения, confusion matrix и примеры предсказаний — в папке `runs/<имя_прогона>/` после обучения (`results.png`, `confusion_matrix.png`, `val_batch*_pred.jpg`).

## Ограничения и дальнейшие шаги

- Основной ограничивающий фактор качества — объём и разнообразие обучающих данных, особенно для `cigarette`, `smoke` и `vape`, а не настройки обучения.
- Дальнейшие направления: расширение датасета (в том числе за счёт открытых источников вроде Roboflow), пересмотр схемы классов (например, объединение `vape` с `cigarette` при сохранении крайне малого числа примеров по вейпу), тестирование на независимой выборке, снятой в условиях, приближённых к реальному применению (дистанция 10–15 м).
